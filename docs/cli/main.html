main¶

dragonfly-openstudio¶

dragonfly openstudio commands.

dragonfly-openstudio [OPTIONS] COMMAND [ARGS]...

Options

--version¶

Show the version and exit.

simulate¶

Commands for simulating URBANopt systems in EnergyPlus.

dragonfly-openstudio simulate [OPTIONS] COMMAND [ARGS]...

system¶

Simulate an URBANopt DES system in EnergyPlus.

Args:
system_file: Path to an URBANopt system parameter file to be simulated
in EnergyPlus. Note that all file paths within the system parameter
must be valid, including the path to the weather file, which will
be used for simulation.
dragonfly-openstudio simulate system [OPTIONS] SYSTEM_FILE

Options

-g, --geojson <geojson>¶

Full path to an URBANopt feature GeoJSON, which can be used to further customize the OpenStudio model. When supplied, the lengths of ThermalConnectors in the loop will be used to account for pipe heat losses.

-sp, --sim-par-json <sim_par_json>¶

Full path to a honeybee energy SimulationParameter JSON that describes all of the settings for the simulation. If None default parameters will be generated.

-f, --folder <folder>¶

Folder on this computer, into which the IDF and result files will be written. If None, the files will be output to a des_energyplus folder in the same directory as the system file.

-log, --log-file <log_file>¶

Optional log file to output the paths of the generated files (osm, idf, sql, rdd, html, err) if successfully created. By default the list will be printed out to stdout

Default:

'-'

Arguments

SYSTEM_FILE¶

Required argument

translate¶

Commands for translating URBANopt systems to OSM/IDF.

dragonfly-openstudio translate [OPTIONS] COMMAND [ARGS]...

system-to-osm¶

Translate an URBANopt system parameter to an OpenStudio Model.

Args:
system_file: Path to an URBANopt system parameter file to be translated
to an OpenStudio model.
dragonfly-openstudio translate system-to-osm [OPTIONS] SYSTEM_FILE

Options

-g, --geojson <geojson>¶

Full path to an URBANopt feature GeoJSON, which can be used to further customize the OpenStudio model. When supplied, the lengths of ThermalConnectors in the loop will be used to account for pipe heat losses.

-sp, --sim-par-json <sim_par_json>¶

Full path to a honeybee energy SimulationParameter JSON that describes all of the settings for the simulation. If None default parameters will be generated.

-osm, --osm-file <osm_file>¶

Optional path where the OSM will be written.

-idf, --idf-file <idf_file>¶

Optional path where the IDF will be written.

-log, --log-file <log_file>¶

Optional log file to output the paths to the generated OSM and IDF files if they were successfully created. By default this will be printed out to stdout

Default:

'-'

Arguments

SYSTEM_FILE¶

Required argument
